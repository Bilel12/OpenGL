                songho Mod rpereira 3 months ago

                It is true that "A square matrix P is projection matrix iif P is idempotent (PP = P)".

                However, the projection matrix in this page is different than in linear algebra definition. The differences are:

                1. OpenGL projection matrix is constructed not only for projection, but also for normalization (NDC), which means the result will be scaled in range of [-1, +1]. If you set left=-1 and right=1, then you will get the same x value for P or P*P. (In this case, our projection matrix is idempotent for x.)

                2. When you project N-dimensional point, the result becomes N-1 dimension in general, for example, a 3D point is projected on a 2D plane. However, OpenGL projection matrix keeps the original dimension even after projection, so we can perform depth test.

                            songho Mod Gallagher Polyn 5 months ago

                            You can use trigonometry to find the width/height. A typical example is to find the optimal camera distance, so an object can fit into the screen.

                            If you know the vertical FOV of your viewing frustum (let half FOV=a) and the bounding radius of an object is r, then the camera distance, d can be computed;

                            d = r / sin(a)

                            And, the height on the screen would be

                            h = r * cos(a)

                            Or, you can find r from d and a.

                                                                    songho Mod Aung KyawOo 9 months ago

                                                                    If you use orthographic projection and want to consistent view without distortion, you need to re-compute the aspect ratio and projection matrix whenever the screen width and height are changed.

                                                                    For instance, if you want to display a 2x2 square perfectly fit to the screen, then try this method:

                                                                    ratio = w / h; 
if(ratio > 1)     // w > h
{
    l = -ratio;
    r =  ratio;
    b = -1;
    t =  1;
}
else
{
    l = -1;
    r =  1;
    b = -1 / ratio;
    t =  1 / ratio;
}

                                                                                                                                      songho Mod AcrylicShrimp a year ago

                                                                                                                                      The viewing frustum actually defines 6 clipping planes (left, right, bottom, top, near and far planes).
                                                                                                                                      To determine if a vertex is in the frustum or not, the vertex will be tested with theses 6 planes. For example, the z-coordinate of a vertex should be between the near plane and the far plane.
                                                                                                                                      Since the equation of the near plane is z + 1 = 0, and the far plane is -z + 1 = 0 in NDC, you can test the z value in NDC (Zn must be above both planes.);

                                                                                                                                       Zn + 1 > 0    Zn > -1 (for near)
                                                                                                                                      -Zn + 1 > 0    Zn <  1 (for far)

                                                                                                                                      Or, in clip space before dividing by Wc (Zn = Zc / Wc);

                                                                                                                                      -Wc < Zc < Wc

                                                                                                                                                -1 in the last row means that X and Y coordinates are depending on -Ze when they are projected on the near plane. (negatively inverse proportional to Ze).

                                                                                                                                                For custom asymmetrical frustum, you need to provide your own left-right-top-bottom-near-far values to construct PROJECTION matrix. You can get a hint from Projection Matrix.
