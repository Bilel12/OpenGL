                                      JoeyDeVries Sensei Jonas gokoon 2 years ago

                                      Yeah it works because we only use a single shader :) so once we call glUseProgram it will remain the active shader until another call to glUseProgram. So even though you first update a uniform before a call to glUseProgram (that will generate an OpenGL error behind the scenes and won't do much) the next time you call glUniform.. it will have the shader activated.

                                      Since this all happens in miliseconds, you won't notice the first frame not properly setting the uniform.

                                      If you want to make sure, you can check OpenGL's function reference pages on glUniform (https://www.opengl.org/sdk/doc... ) where it states:
                                      "glUniform operates on the program object that was made part of current state by calling glUseProgram."

                                                                                                                                        JoeyDeVries Sensei Gabriel Sussumu 2 years ago

                                                                                                                                        I think they need different gl_Position which can only be achieved by having different shaders.

                                                                                                                                        The position of your objects is determined by whatever you place in gl_Position. Since both your objects are identical, they will pass in the same positions to gl_Position which will result in both your objects being rendered at the same position and thus only one becomes visible. It doesn't matter how many shaders you use for that purpose, as long as their positions are identical, you won't see a difference.

                                                                                                                                        To actually render different objects to different positions you either want to:
                                                                                                                                        1. Update the vertex data for the other objects (or create different vertex data specifically for those objects) and create/update the VBOs.
                                                                                                                                        2. Add an offset to their position via a uniform variable (probably enough for your case at the moment).
                                                                                                                                        3. OR (the most flexible solution) use transformation matrices. We'll get to those in the 'Transformations' tutorial.

                                                                                                                                        If you're still not sure how it should work I suggest to work your way through the tutorials and eventually you'll get the hang of it (especially the transformations tutorial where we render 10 objects in the scene, each at a different position/rotation)

                                                                                                                                                          JoeyDeVries Sensei Baenor 2 years ago

                                                                                                                                                          The stride defines per vertex attribute, where the next value of that specific vertex attribute is in your vertex data.

                                                                                                                                                          For example, with your set of data, the first two are vertex positions so if we want to get the next first vertex position in your vertex data, we need to move 5 times the size of a GLfloat into the vertex array to get the next one. This is the stride.
                                                                                                                                                          Since the size of each vertex is 5 floats, the stride should become 5 * sizeof(GLfloat).

                                                                                                                                                          Another example: The first color attribute (red component) is the third float of each vertex which is at position 2 * sizeof(GLfloat) in the array. The next specific instance of this red color float is then at 2 * sizeof(GLfloat) + stride ==> 2 * sizeof(GLfloat) + 5 * sizeof(GLfloat) ==> 7 * sizeof(GLfloat).

                                                                                                                                                          I hope these examples give you more insight into the stride parameter. If you still find it difficult, you'll find more examples of the stride parameter throughout the tutorials.
