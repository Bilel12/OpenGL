                                        I'm no expert on GLM since I've been following this tutorial in C and have been writing my own set of vector and matrix functions using __attribute__((vector_size())) vectors and matrices. But glm::lookAt() looks a bit like gluLookAt from back when glu was pretty popular.

                                        gluLookAt took a few parameters, the position of the camera, the point being looked at and the up vector.

                                        In your example, if the three parameters are as I have described them, then you would be looking from point (0, 0, 2) down onto point (0, 0, 0) (the origin) and your up vector would be (0, 0, 1).

                                        I hope you can see problems would arise calculating the desired vector if you set "up" to be going directly behind you.

                                        Try glm::lookAt(glm::vec3(0, 0, 2), glm::vec3(0, 0, 0), glm::vec3(0, 1, 0)) instead.
